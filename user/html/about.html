Project Highlights

We achieved an app‚Äëlike experience with no frameworks. Components share a consistent visual language and switch between light/dark modes via a single html.dark toggle. Performance remains excellent because everything is static and minimal.

The cart is reactive on the client: quantities, coupons (SAVE10, FREESHIP), shipping options, and tax calculations update totals instantly and persist per user in localStorage. The navbar badge stays in sync using a tiny event bus (bh:cart-changed).

Accessibility guided many decisions: labelled controls, keyboard‚Äëoperable steppers, visible focus states, and logical document structure. The layout adapts gracefully to small screens while keeping the order summary visible on larger displays.

Challenges Faced

Keeping session state and cart UI consistent across pages and tabs was challenging. We solved this by centralising state in app.js, listening to storage events, and dispatching custom events to trigger page re‚Äërenders when needed.

Designing a dark theme with strong contrast while preserving a soft look required careful color and border tuning. We added precise overrides (e.g., quantity input text, cart badge) to prevent legibility issues.

Finally, we balanced flexibility with structure by documenting class names, avoiding heavy global resets, and keeping components small so pages could evolve without style conflicts.
